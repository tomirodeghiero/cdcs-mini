==============================================================
cdcs — ejemplos para correr
==============================================================

Cada bloque es un archivo Python que podés pegar en
examples/<nombre>.py y procesar con:

    uv run cdcs examples/<nombre>.py

Para ver el JSON pretty:

    uv run cdcs examples/<nombre>.py | jq .

Para guardar el reporte:

    uv run cdcs examples/<nombre>.py --out report.json

Exit codes:
    0 = limpio  ·  1 = diagnósticos detectados  ·  2 = error de uso

EJEMPLO 8 — error ContradictoryExamplesError
archivo: examples/08_contradictory.py
exit code esperado: 1

Nota: la DSL de behavior NO acepta bloques Python (if/else, for,
asignaciones). Cada paso es: operación (expresión), `require ...`
o `return ...`. Para casos con ramas, expresá la condición en
`require` o usá una expresión condicional dentro del return.
==============================================================
def absolute(n: int) -> int:
    """@generate
    behavior:
      return -n if n < 0 else n

    examples:
      absolute(-5) == 5
      absolute(-5) == 10        # <-- mismos args, distinto resultado
      absolute(3) == 3

    constraints:
      no_imports
    """
    ...

class TokenService:
    def issue(self, user_id: int, ttl_seconds: int) -> str:
        """@generate
        behavior:
          require ttl_seconds > 0
          return self._sign(str(user_id))

        examples:
          issue(42, 60) == "signed:42"
          issue(1, 0) raises ValueError

        calls:
          self._sign(payload: str) -> str    # HMAC del payload
          self._now() -> int                  # epoch seconds

        reads:
          self.secret_key: bytes

        constraints:
          no_network
          no_filesystem

==============================================================
CÓMO USAR ESTOS EJEMPLOS
==============================================================

1) Crear los archivos en examples/ (o copiar y pegar cualquier
   bloque en un .py nuevo).

2) Procesar uno:

       uv run cdcs examples/01_parse_port.py

3) Procesar todos y ver el código de salida:

       for f in examples/*.py; do
         echo "=== $f ==="
         uv run cdcs "$f" > /dev/null
         echo "exit=$?"
       done

4) Usar el backend HTTP:

       uv run uvicorn web.backend.app.main:app --reload &
       curl -s -X POST http://127.0.0.1:8000/reports/from-file \
            -F "file=@examples/01_parse_port.py" | jq .

5) Modo synthesizer (requiere extra `synthesis` y un LLM):

       uv sync --extra synthesis
       export ANTHROPIC_API_KEY=sk-...    # o usar Ollama / Pollinations
       uv run cdcs compile examples/10_token_service.py --dest examples/_gen/
       uv run cdcs check   examples/10_token_service.py --dest examples/_gen/
